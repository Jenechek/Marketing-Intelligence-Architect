# Marketing Intelligence

Marketing Intelligence — локальное веб-приложение для интернет-маркетолога. Приложение запускается на компьютере, показывает русскоязычный список сайтов, сохраняет добавленные сайты в SQLite и ведёт локальный журнал.

Сейчас доступны добавление, редактирование, подтверждаемое окончательное удаление сайта, безопасная ручная проверка одной стартовой страницы и фоновый полный обход внутренних ссылок. Ход полного обхода показывается на отдельном автоматически обновляемом экране, а итог, метаданные страниц и снимки успешно разобранного содержимого сохраняются в локальной SQLite. Аналитика ещё не реализована.

## Требования к окружению

- Windows 10 или Windows 11;
- Python 3.11 или новее;
- Яндекс Браузер, Google Chrome или Mozilla Firefox;
- интернет для первой установки Python-зависимостей.

## Быстрый запуск на Windows

Откройте PowerShell в папке проекта и последовательно выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[test]"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

Откройте [http://127.0.0.1:8000](http://127.0.0.1:8000) в поддерживаемом браузере.

На странице заполните поля «Название» и «URL», затем нажмите «Добавить сайт». Сайт появится в списке и останется доступен после перезапуска приложения.

Действие «Проверить доступность» сначала проверяет `robots.txt` и только при разрешении запрашивает одну стартовую страницу. Успешная HTML-страница потоково читается в пределах 2 МиБ; количество найденных ссылок видно сразу, а список из не более чем 200 уникальных внутренних ссылок закрыт по умолчанию и раскрывается по запросу. Проверка не переходит по найденным ссылкам, не выполняет JavaScript и не отправляет формы. HTML и ссылки не сохраняются. Начало, завершение, результат, сообщение и полученные HTTP-коды сохраняются в локальной SQLite и остаются в истории после перезапуска. При окончательном удалении сайта экран подтверждения показывает число связанных записей; сайт и его история удаляются вместе.

Действие «Полный обход» запускает последовательный обход в фоне с безопасными значениями: не более 200 страниц, глубина 3, задержка 1 секунда и ожидание ответа 15 секунд. Эти значения и User-Agent можно изменить в закрытом по умолчанию блоке «Расширенные настройки»; десятичные значения принимаются с точкой или запятой. При ошибке блок раскрывается, введённые значения сохраняются, а запуск не создаётся. Во время работы остальные экраны доступны, страница запуска показывает живые счётчики и позволяет раскрыть фактически использованные настройки. Одновременно выполняется только один полный обход. Сайт с выполняющимся обходом удалить нельзя: экран удаления предлагает дождаться завершения и перейти к активному запуску. После завершения обычное подтверждённое удаление снова доступно. Для успешных HTML-страниц полный обход сохраняет время проверки, Title, Description, H1, нормализованный текст, SHA-256 и полный список внутренних ссылок без сырого HTML. Частично завершённый обход сохраняет успешные страницы и ошибки, не удаляя данные прежних запусков.

Подробная инструкция, включая решение частых ошибок: [`docs/WINDOWS.md`](docs/WINDOWS.md).

## Остановка

В окне PowerShell нажмите `Ctrl+C`. Затем при необходимости выйдите из виртуального окружения:

```powershell
deactivate
```

## Тестирование

```powershell
python -m pytest
```

Тест создаёт базу только во временной папке и не затрагивает рабочие данные.

## Локальные данные

При первом запуске автоматически создаются:

- `data/marketing_intelligence.db` — локальная база SQLite;
- `logs/marketing-intelligence.log` — журнал работы приложения.

Папки `data`, `logs`, базы, журналы, секреты и виртуальное окружение исключены из Git.

Для отдельного расположения можно задать переменные окружения до запуска:

```powershell
$env:MI_DATA_DIR = "D:\Marketing Intelligence\data"
$env:MI_LOGS_DIR = "D:\Marketing Intelligence\logs"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

Для точного адреса SQLite предназначена переменная `MI_DATABASE_URL`. Обычному пользователю она не требуется.

```text
marketing-intelligence/
├── src/marketing_intelligence/  # исходный код приложения
│   ├── static/                  # стили интерфейса
│   └── templates/               # серверные HTML-шаблоны

```

Приложение построено как минимальный модульный монолит: конфигурация, база, журналирование и веб-слой разделены, но запускаются одним процессом.

## Текущие ограничения

- ручная проверка ограничена `robots.txt` и одной стартовой страницей; для HTML читается не более 2 МиБ и показывается не более 200 внутренних ссылок без перехода по ним;
- сохранённые пресеты, настройки сайта, расписание и отдельный список истории запусков ещё не реализованы;
- приложение рассчитано на одного локального пользователя;
- доступ из интернета не настраивается;
- пользовательская проверка на Windows пройдена в Яндекс Браузере, Google Chrome и Mozilla Firefox.
