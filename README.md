# Marketing Intelligence

Marketing Intelligence — локальное веб-приложение для интернет-маркетолога. Приложение запускается на компьютере, показывает русскоязычный список сайтов, сохраняет добавленные сайты в SQLite и ведёт локальный журнал.

Сейчас доступны добавление, редактирование, подтверждаемое окончательное удаление сайта, безопасная ручная проверка одной стартовой страницы, фоновый полный обход внутренних ссылок и автоматические полные обходы по расписанию. Ход полного обхода показывается на отдельном автоматически обновляемом экране, а итог, метаданные страниц, снимки успешно разобранного содержимого и надёжно обнаруженные цены сохраняются в локальной SQLite. Между последовательными успешно завершёнными обходами приложение строит историю изменений страниц и достоверно обнаруженных цен с фильтрами, точными подробностями, ручным статусом просмотра и экспортом в JSON или CSV. По последнему пригодному обходу также доступна функциональная карта структуры сайта. Для выбранного сайта можно вручную импортировать показатели страниц из Google Search Console `Pages.csv`.

## Требования к окружению

- Windows 10/11, macOS или Linux;
- Python 3.11 или новее;
- современный настольный браузер: Яндекс Браузер, Google Chrome, Microsoft Edge, Mozilla Firefox или Safari;
- интернет для первой установки Python-зависимостей.

Практический запуск и пользовательская проверка пока выполнены только на Windows. Инструкции для macOS и Linux составлены по стандартному способу запуска Python-проектов, но ещё не проверены на реальных системах.

## Быстрый запуск на Windows

Откройте PowerShell в папке проекта и последовательно выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e ".[test]"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

## Быстрый запуск на macOS и Linux

> Эта инструкция ещё не проверена на реальных системах macOS и Linux.

Откройте Terminal в папке проекта и последовательно выполните:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e ".[test]"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

После запуска откройте [http://127.0.0.1:8000](http://127.0.0.1:8000) в поддерживаемом браузере.

На странице заполните поля «Название» и «URL», затем нажмите «Добавить сайт». Сайт появится в списке и останется доступен после перезапуска приложения.

Действие «Проверить доступность» сначала проверяет `robots.txt` и только при разрешении запрашивает одну стартовую страницу. Успешная HTML-страница потоково читается в пределах 2 МиБ; количество найденных ссылок видно сразу, а список из не более чем 200 уникальных внутренних ссылок закрыт по умолчанию и раскрывается по запросу. Проверка не переходит по найденным ссылкам, не выполняет JavaScript и не отправляет формы. HTML и ссылки не сохраняются. Начало, завершение, результат, сообщение и полученные HTTP-коды сохраняются в локальной SQLite и остаются в истории после перезапуска. При окончательном удалении сайта экран подтверждения показывает число связанных записей; сайт и его история удаляются вместе.

Действие «Полный обход» запускает последовательный обход в фоне с безопасными значениями: не более 200 страниц, глубина 3, задержка 1 секунда и ожидание ответа 15 секунд. Эти значения и User-Agent можно изменить в закрытом по умолчанию блоке «Расширенные настройки»; десятичные значения принимаются с точкой или запятой. При ошибке блок раскрывается, введённые значения сохраняются, а запуск не создаётся. Во время работы остальные экраны доступны, страница запуска показывает живые счётчики и позволяет раскрыть фактически использованные настройки. Одновременно выполняется только один полный обход. Сайт с выполняющимся обходом удалить нельзя: экран удаления предлагает дождаться завершения и перейти к активному запуску. После завершения обычное подтверждённое удаление снова доступно. Для успешных HTML-страниц полный обход сохраняет время проверки, Title, Description, H1, нормализованный текст, SHA-256 и полный список внутренних ссылок без сырого HTML. Цены сохраняются отдельно, если они надёжно обнаружены в JSON-LD или Microdata Schema.org; отсутствие такой разметки означает «цена надёжно не обнаружена», а не отсутствие цены на странице. Частично завершённый обход сохраняет успешные страницы, цены и ошибки, не удаляя данные прежних запусков.

## Автоматические обходы по расписанию

Действие «Расписание» позволяет включить для сайта один ежедневный или еженедельный полный обход. По умолчанию расписание выключено; безопасное начальное правило — раз в неделю в 09:00 по локальному времени. Первый запуск всегда назначается на будущее. Настройки полного обхода можно изменить в закрытом блоке расширенных параметров.

Расписание, очередь и журнал сохраняются в SQLite. Одновременно выполняется только один полный обход во всём локальном приложении, поэтому наступившие проверки разных сайтов и ручные повторы ожидают своей очереди. Журнал показывает результат, связанную страницу обхода и кнопку повтора для частично завершённых, отложенных, ошибочных или прерванных запусков.

Планировщик работает, пока запущен процесс FastAPI; браузер можно закрыть. Если процесс был остановлен, приложение после следующего запуска зафиксирует пропущенные периоды одной записью, но не станет выполнять их задним числом. Полностью автономный серверный режим относится к этапу 14.

### Необязательные почтовые уведомления

Без SMTP расписание, очередь, журнал и ручной повтор работают полностью. Для уведомлений задайте переменные окружения перед запуском приложения:

| Переменная | Значение |
| --- | --- |
| `MI_SMTP_HOST` | адрес SMTP-сервера |
| `MI_SMTP_SECURITY` | `starttls` или `ssl`; по умолчанию `starttls` |
| `MI_SMTP_PORT` | порт; по умолчанию 587 для STARTTLS и 465 для SSL |
| `MI_SMTP_USERNAME` | необязательный логин; задаётся только вместе с паролем |
| `MI_SMTP_PASSWORD` | необязательный пароль; не сохраняется в БД и не показывается в интерфейсе |
| `MI_SMTP_FROM` | один адрес отправителя |
| `MI_SMTP_TO` | один адрес получателя |
| `MI_SMTP_TIMEOUT` | ожидание от 1 до 60 секунд; по умолчанию 10 |

Письмо отправляется только для частично завершённого, отложенного, ошибочного или прерванного запуска. Ошибка отправки отмечается отдельно и не меняет результат обхода. Конкретные настройки нужно получить у своего почтового провайдера; фактическая отправка через Gmail, Outlook, Яндекс или другого внешнего поставщика без пользовательских реквизитов не проверялась.

## Импорт Google Search Console

Действие «Импорт данных» открывает загрузку CSV отчёта Google Search Console «Эффективность → Страницы». Перед загрузкой в Search Console выберите вкладку «Страницы», уберите дополнительные фильтры и запомните начало и конец периода: эти даты нужно указать вручную в приложении.

Поддерживаются CSV в UTF-8 с BOM и без него, разделённые запятой, точкой с запятой или табуляцией. Обязательны страница, клики и показы; средняя позиция необязательна. После загрузки приложение предлагает сопоставление русских или английских заголовков и показывает безопасный предпросмотр. Данные записываются только после подтверждения. Исходный файл не сохраняется, а неподтверждённый предпросмотр исчезает через 30 минут или после перезапуска приложения.

Повторный импорт того же сайта, периода и URL обновляет изменившиеся показатели без создания дублей. Строки, отсутствующие в новом файле, не удаляются автоматически, поскольку выгрузка Search Console может быть неполной или отфильтрованной. Таблица показывает показы, клики, рассчитанный CTR, среднюю позицию и наличие страницы в последнем завершённом или частично завершённом обходе. Отсутствие в обходе не доказывает, что страница сиротская.

Один файл ограничен 5 МиБ, 10 000 строками данных и 50 столбцами. Импорт `Pages (Indexing).csv`, отчёты Яндекс Метрики, XLS/XLSX и API пока не поддерживаются.

## Карта структуры сайта

Действие «Структура» открывает карту наиболее нового завершённого или частично завершённого полного обхода выбранного сайта. Частичный результат и достигнутые ограничения явно отмечаются, чтобы пользователь не принял неполную карту за полную.

Карта показывает список и дерево страниц, функциональный граф внутренних ссылок, глубину, число входящих и исходящих связей, битые и непроверенные ссылки. Для HTML-страниц рассчитываются PageRank и место в текущей карте, а также сигналы недостаточной связности, тупиковых зон, структурных островов, циклических ловушек, аномальной глубины и бутылочных горлышек. Это расчётные показатели текущего обхода, а не доказанная важность страницы или автоматическая рекомендация.

Список можно фильтровать по URL, глубине, результату обработки, наличию проблемных ссылок и структурному сигналу; фильтры и страница сохраняются при переходе в подробности. Битой считается только фактически запрошенная внутренняя ссылка с HTTP-кодом 400–599. Запрет `robots.txt`, перенаправление, timeout, ошибка сети, корректный non-HTML ресурс и не проверенная из-за лимита ссылка показываются как отдельные состояния.

Граф не выводит более 100 узлов одновременно: для крупной структуры сначала применяется фильтр. Просмотр карты и подробностей не изменяет сохранённые данные.

## История изменений

Первый успешно завершённый полный обход сайта становится базовой линией и не создаёт событий. После следующего успешно завершённого обхода приложение сравнивает последовательные снимки и показывает новые и удалённые страницы, изменения Title, Description, H1, текста и внутренних ссылок, а также достоверные изменения одной цены или диапазона на однозначной товарной посадочной странице. Частичные и ошибочные обходы базовую линию не меняют.

Историю можно открыть для одного сайта или сразу для всех сайтов, отфильтровать по сайту, типу, датам и ручному статусу просмотра. Открытие подробностей само не отмечает событие просмотренным: статус меняется только явным действием и может быть отменён. JSON и CSV экспортируют всю текущую отфильтрованную выборку, а не только открытую страницу; CSV подготовлен для безопасного открытия в Excel.

Подробная инструкция, включая решение частых ошибок: [`docs/WINDOWS.md`](docs/WINDOWS.md).

В окне PowerShell или терминала, где работает приложение, нажмите `Ctrl+C`. Затем при необходимости выйдите из виртуального окружения:

```powershell
python -m pytest
```

## Локальные данные

При первом запуске автоматически создаются:

- `data/marketing_intelligence.db` — локальная база SQLite;
- `logs/marketing-intelligence.log` — журнал работы приложения.

Папки `data`, `logs`, базы, журналы, секреты и виртуальное окружение исключены из Git.

Для отдельного расположения можно задать переменные окружения до запуска:

```powershell
$env:MI_DATA_DIR = "D:\Marketing Intelligence\data"
$env:MI_LOGS_DIR = "D:\Marketing Intelligence\logs"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

В macOS или Linux используются равнозначные команды:

```bash
export MI_DATA_DIR="$HOME/marketing-intelligence-data"
export MI_LOGS_DIR="$HOME/marketing-intelligence-logs"
python -m uvicorn marketing_intelligence.main:app --host 127.0.0.1 --port 8000
```

Для точного адреса SQLite предназначена переменная `MI_DATABASE_URL`. Обычному пользователю она не требуется.

## Текущие ограничения

- ручная проверка ограничена `robots.txt` и одной стартовой страницей; для HTML читается не более 2 МиБ и показывается не более 200 внутренних ссылок без перехода по ним;
- отдельные сохраняемые пресеты настроек обхода вне расписаний ещё не реализованы;
- автоматические обходы не выполняются при полностью остановленном процессе FastAPI и не запускаются задним числом после простоя;
- приложение рассчитано на одного локального пользователя;
- доступ из интернета не настраивается;
- пользовательская проверка на Windows пройдена в Яндекс Браузере, Google Chrome и Mozilla Firefox;
- инструкции запуска на macOS и Linux ещё не проверены на реальных системах; подтверждённая совместимость будет зафиксирована только после фактической проверки.
