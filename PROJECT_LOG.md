## Обязательное правило после завершения задачи

После каждой успешно выполненной и проверенной задачи необходимо:

1. обновить `TASKS.md`;
2. обновить `PROJECT_LOG.md`;
3. при необходимости обновить `ROADMAP.md`;
4. при необходимости обновить `CHANGELOG.md`;
5. если принято новое решение — обновить `DECISIONS.md`;
6. если изменились правила проекта — обновить `PROJECT.md`.

Задача не считается завершённой, пока обязательные документы не приведены в актуальное состояние.

## Запрет на пропуск задач

Задачи выполняются только в утверждённой последовательности.

Нельзя:

- начинать следующую задачу до завершения и проверки текущей;
- пропускать незавершённую задачу ради более поздней;
- отмечать задачу завершённой без проверки результата;
- менять порядок задач без явного обновления `TASKS.md` и, при необходимости, `ROADMAP.md`.

Если задача заблокирована, блокировка фиксируется в `TASKS.md`. После этого план изменяется только отдельным явным решением.
