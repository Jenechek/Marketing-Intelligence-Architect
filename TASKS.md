## Ожидаемый результат

- из списка сайтов открывается отдельный экран полного обхода с одним основным действием и безопасными настройками по умолчанию;
- защищённый POST создаёт `CrawlRun` до ответа, запускает отслеживаемую фоновую задачу и отвечает перенаправлением 303 на экран запуска;
- во время `running` другие экраны доступны, текущие счётчики сохраняются и показываются с автоматическим обновлением;
- записи страниц публикуются только вместе с атомарным завершением запуска;
- одновременно выполняется только один полный обход, повторный запрос ведёт к активному запуску без новой записи и сетевого вызова;
- shutdown безопасно останавливает фоновые задачи, а следующий startup переводит незавершённый запуск в `interrupted`;
- неизвестные идентификаторы и неверный action token не создают записей и не вызывают сеть;
- расширенные настройки, список страниц, журнал ошибок, отмена, расписание, снимки и аналитика не добавляются.

## Критерии завершения

- run существует до ответа 303, а блокируемый crawler не мешает работе других экранов;
- во время `running` видны актуальные `processed/max_pages` и доступные счётчики, после завершения — итог и признак ограничения;
- до финальной транзакции нет записей страниц;
- повторный запуск, неверный токен и неизвестные идентификаторы не вызывают сеть и не создают run;
- незавершённый запуск после следующего startup становится `interrupted`;
- полная регрессия и фактическая loopback-проверка проходят;
- независимая приёмка по поручению пользователя подтверждена.
